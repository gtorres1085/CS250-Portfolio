# CS 250 Portfolio – Software Development Lifecycle (Agile)

## Overview
This repository contains my work for CS 250, where I explored the Software Development Life Cycle (SDLC) using the Scrum-Agile framework. Throughout the course, I worked on the SNHU Travel project and took on different Scrum roles to understand how Agile teams operate in a real-world environment.

## Project Description
The main project focused on developing a niche travel booking system for SNHU Travel. The goal was to design a system that allows users to browse and book unique vacation experiences while adapting to changing requirements and priorities.

## Agile & Scrum Experience
During this course, I worked in multiple roles within the Scrum Team:

- **Scrum Master**
  - Facilitated Scrum events (Sprint Planning, Daily Scrum, Review, Retrospective)
  - Helped remove blockers and improve team communication

- **Product Owner**
  - Created and prioritized user stories
  - Focused on delivering value based on user and stakeholder needs

- **Developer**
  - Implemented features based on changing requirements
  - Collaborated with the team to ensure alignment and functionality

- **Tester**
  - Developed test cases using acceptance criteria
  - Worked closely with the Product Owner to clarify requirements and improve quality

## Key Agile Practices Used
- Sprint Planning
- Daily Scrums
- Sprint Reviews
- Sprint Retrospectives
- Backlog Refinement
- Continuous Testing
- Use of User Stories and Acceptance Criteria

## Tools & Technologies
- Agile Scrum Framework
- Azure Boards (project tracking and backlog management)
- GitHub (version control and portfolio management)

## Key Takeaways
- Agile allows teams to adapt quickly to changing requirements
- Communication is critical for team success and product quality
- Breaking work into smaller tasks improves efficiency and clarity
- Continuous feedback helps avoid major issues later in development
